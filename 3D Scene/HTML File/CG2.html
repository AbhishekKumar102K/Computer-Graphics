Creating a 3D Scene Using OpenGL Transformations, matrix stack operations and 3D viewing pipeline

Introduction

In this Assignment we have tried to create a 3D scene of an Auditorium using OpenGL Transformations, matrix stack operations, and 3D viewing pipeline. We have also used Blender to create objects like Speakers, Mic and Podium and then exported them into our scene.

Classes Used And Their Brief Description

We used various classes and functions to make our code look more organised and easy to understand. Here are all the classes that we used to make this Auditorium and a brief description about them.

  • Audi Class :

    This class creates the basic structure of the auditorium including all the walls, the entrance and the stage of the Auditorium.

  • Blender_imp Class :

    This Class is used to import all the blender objects into the Auditorium. It includes functions like make_speakers(), make_podium() and make_mic().

  • Chair Class :

    This Class is used to make the seats of the Auditorium.

  • Exitgate Class :

    This Class makes the Exits of the auditorium.

  • Light Class :

    This Class makes the Ceiling lights of the Auditorium.

  • Reader Class :

    This Class is responsible for reading the object files created by blender and then finally drawing these objects using glBegin(GL_TRIANGLE)function.

  • Cuboid Class :

    This Class makes the Cubes, Cuboids for the scene.

  • Stair Class :

    This Class makes the floor steps of the auditorium.

Controls Of The Scene

  • 'W' : Rotate The camera about X-axis in Counter-Clockwise Sense
  • 'S' : Rotate The camera about X-axis in Clockwise Sense
  • 'A' : Rotate The camera about Y-axis in Counter-Clockwise Sense
  • 'D' : Rotate The camera about Y-axis in Clockwise Sense
  • 'Q' : Rotate The camera about Z-axis in Counter-Clockwise Sense
  • 'E' : Rotate The camera about Z-axis in Clockwise Sense
  • 'O' : Move The camera upwards
  • 'L' : Move The camera downwards
  • ';' : Move The camera Right
  • 'K' : Move The camera Left
  • '=' : Move The camera BackWards
  • '-' : Move The camera Inwards
  • 'R' : Reset The camera to the initial position

Implementaion Issues

  • Importing complex shapes and objects from Blender leads to memory overflow. It also increases the time complexity of the program because of having to draw many polygons. Thus, we have imported only simple and basic objects.
  • Due to lack of shadows and lighting, the solid color given to the objects makes them look like one polygon instead of a 3D object. You may notice this in the Podium which looks like a 2d Outline of an object as it loses its identity as a 3d podium.
  • The shape of the floor of the Auditorium is trapezium due to which we couldn't arrange Chairs properly one behind the other and mantain uniformity. This lead to the chair arrangement looking visually unaesthetic.
